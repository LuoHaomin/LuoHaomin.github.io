## 设计原则

### 1. 渐进式学习

- **从基础到高级**：按照从简单到复杂的顺序组织内容
- **循序渐进**：每个部分都建立在前一部分的基础上
- **清晰路径**：提供明确的学习路径指南

### 2. 模块化设计

- **独立模块**：每个部分都是相对独立的模块
- **清晰边界**：每个模块有明确的边界和职责
- **易于导航**：通过索引文件轻松导航

### 3. 完整性

- **全面覆盖**：覆盖 Bevy 的核心功能和高级特性
- **实用示例**：每个概念都配有实际代码示例
- **最佳实践**：分享实际开发中的经验和技巧

### 4. 可维护性

- **统一格式**：所有文档遵循统一的格式规范
- **清晰结构**：每个文档都有清晰的结构
- **易于更新**：结构设计便于后续更新和维护

## 文档规范

### 文件命名

- 使用中文文件名
- 使用连字符或下划线分隔单词
- 保持文件名简洁明了

### 文档结构

每个文档应该包含（按照新架构标准）：

1. **概述**：学习目标、前置知识要求
2. **核心概念**：概念解释、设计思想、为什么需要这个功能
3. **基础用法**：最简单的使用示例、代码示例（标注来源文件）、关键要点说明
4. **进阶用法**（可选）：更复杂的使用场景、代码示例（标注来源文件）、注意事项和最佳实践
5. **实际应用**（可选）：在游戏开发中的应用场景、常见问题和解决方案、性能考虑
6. **相关资源**：相关源代码文件、官方文档链接、进一步学习建议

### 代码示例

- 使用 Rust 代码块
- 添加必要的注释
- 确保代码可以运行
- 使用有意义的变量名

### 格式规范

- 使用 Markdown 格式
- 使用清晰的标题层级
- 使用列表和表格组织内容
- 添加必要的链接
